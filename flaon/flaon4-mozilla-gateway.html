flaON - migration to Mozilla Gateway

These days I am doing a few changes to my flaON project and I'm talking about moving from openhab2 to Mozilla Gateway (referenced below as MG).

During the Eastern days from this year I've discovered the MG. Not a WOW thing when it comes about layout interface, but impresive about development and possibilities.

Unlike many smart home hubs and connected IoT devices on the market, in the MG data is not stored or processed in the cloud. It stays on the gateway. The gateway can often bypass the need for purchasing an IoT hub for each brand and instead of downloading a different app for each brand, I can manage everything from one place.

But I mostly was impressed by configurable addons and extensions. I managed to build an MQTT extension for my gateway, which allows me to have an overview of the mqtt communication in my home.

After that, it comes with webthings framework, which allows me to build my own web things and monitor or control them over the web, with good libraries in Python and Java.
